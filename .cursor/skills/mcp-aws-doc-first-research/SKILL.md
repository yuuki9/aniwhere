# AWS Documentation First Research

## Policy

- Prefer official documentation evidence before giving implementation guidance.
- Use focused section reads for accuracy.
- Distinguish documented facts from assumptions.

## Execution Flow

1. Search documentation with a precise phrase and intent.
2. Read only relevant sections.
3. Return:
   - key facts
   - limits/constraints
   - links to sources
4. If needed, hand off validated assumptions to AWS API MCP checks.
